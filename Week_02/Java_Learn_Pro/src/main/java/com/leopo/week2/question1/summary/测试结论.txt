1.并行GC的吞吐量比较大，略微优于串行GC,明显优于CMS，但是并未比G1优秀；
2.堆较小的时候，基本每种GC策略都发生了OOM和FULLGC死循环；
3.串行GC随着堆的增加，吞吐下降；但是并行GC和G1都是上升的，CMS似乎也是上升的，其中，G1最为优先。
4.从数据看，CMS和G1的暂停时间是最为优秀的。
5.总体来看，并行GC的单位时间的响应数是最少的，G1是比较优秀，CMS次一点。


总体来看：
1.串行GC的吞吐随着堆的增大，吞吐下降的还是比较明显，因此不太适合当前市面内存的工作。堆小的时候，吞吐还可以。
2.并行GC吞吐较好，但是压测的时候发现响应比较小。
3.CMS的吞吐比并行GC小，但是响应比GC好一些，更适合线上响应要求高的场景。
4.G1的吞吐和响应都比较好，而且堆大的情况下更优秀。
